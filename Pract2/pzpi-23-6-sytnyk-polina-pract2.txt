2 ЗАВДАННЯ
Метою практичного заняття є дослідження архітектури складної програмної системи Telegram та аналіз застосованих у ній архітектурних підходів, принципів побудови, взаємодії компонентів, механізмів масштабування та забезпечення надійності функціонування.
Актуальність вибору цієї системи зумовлена тим, що Telegram є однією з найвідоміших сучасних платформ обміну повідомленнями, яка поєднує клієнтські застосунки, серверну інфраструктуру, засоби передавання мультимедійного контенту, механізми безпечного зв’язку та системи обслуговування великої кількості користувачів.
У межах виконання роботи необхідно проаналізувати програмну систему Telegram як приклад складної багатокомпонентної платформи, визначити її призначення, основні функціональні можливості та коло користувачів. Також потрібно дослідити архітектурний стиль системи, описати її основні компоненти, зокрема клієнтський рівень, серверну частину, підсистеми оброблення повідомлень, служби доставки медіафайлів, механізми автентифікації та API для ботів.
Окрему увагу в роботі слід приділити оцінюванню масштабованості та надійності архітектурних рішень Telegram, зокрема балансуванню навантаження, кешуванню, розподіленому зберіганню даних і забезпеченню відмовостійкості.


3 ОПИС ВИКОНАНОЇ РОБОТИ
3.1 Загальні відомості про систему
Об’єктом дослідження в межах практичного заняття обрано програмну систему Telegram — одну з найбільших світових платформ для миттєвого обміну повідомленнями, передавання файлів, організації групового спілкування та масового поширення контенту.
Система орієнтована як на звичайних користувачів, які використовують приватні та групові чати, так і на адміністраторів каналів, компанії та розробників ботів.
Telegram підтримує мобільні, десктопні та вебверсії, які синхронізуються між собою в реальному часі.

3.2 Архітектурний стиль системи
Архітектура Telegram може бути охарактеризована як багаторівнева розподілена клієнт-серверна архітектура з елементами мікросервісного підходу.
На клієнтському рівні функціонують мобільні застосунки, вебверсія та десктопні клієнти. На серверному рівні працюють сервіси автентифікації, маршрутизації повідомлень, керування сесіями, зберігання медіафайлів та аналітики.
Такий підхід забезпечує можливість незалежного масштабування окремих компонентів, високу доступність сервісу та стабільну роботу при великій кількості одночасних користувачів.

3.3 Основні компоненти системи
До основних компонентів Telegram належать:
•	клієнтські застосунки; 
•	API Gateway; 
•	сервери автентифікації; 
•	сервери повідомлень; 
•	сервери зберігання даних; 
•	медіа-сервери; 
•	CDN-інфраструктура; 
•	Telegram Bot API. 
Клієнтський рівень забезпечує взаємодію користувача із системою. Серверна частина приймає запити, перевіряє права доступу та виконує доставку повідомлень. Медіа-сервери відповідають за передавання фото, відео та документів.

3.4 Взаємодія компонентів
Взаємодія компонентів у Telegram відбувається як послідовність запитів між клієнтським застосунком, серверною інфраструктурою та сховищами даних.
Коли користувач надсилає повідомлення, клієнт передає його через API Gateway. Після перевірки користувача сервер повідомлень маршрутизує дані до адресата.
Якщо передається файл або медіа, дані зберігаються окремо та доставляються через CDN.
Усі клієнтські пристрої користувача автоматично синхронізують історію листування.

3.5 Масштабованість та надійність
Масштабованість Telegram забезпечується завдяки горизонтальному масштабуванню, sharding та replication.
Горизонтальне масштабування означає можливість додавати нові сервери без зупинки системи.
Sharding дозволяє розподіляти дані між кількома базами даних, а replication — створювати резервні копії на інших вузлах.
Надійність підвищується завдяки балансуванню навантаження, резервуванню компонентів та автоматичному переключенню у разі збоїв.

3.6 Інфраструктура системи
Telegram використовує дата-центри в різних регіонах світу. Це дає змогу зменшити затримки та забезпечити швидкий доступ користувачів до сервісу.
Для доставки фото, відео та файлів використовується CDN — розподілена мережа доставки контенту.
Також застосовуються балансувальники навантаження, які рівномірно розподіляють трафік між серверами.

3.7 Безпека та протокол MTProto
Важливим елементом архітектури Telegram є власний протокол MTProto.
Він використовується для захищеної передачі даних між клієнтом і сервером.
Протокол поєднує високу швидкість роботи та криптографічний захист. Для шифрування використовуються сучасні алгоритми, зокрема AES та RSA.
Для секретних чатів застосовується наскрізне шифрування, що забезпечує додатковий рівень конфіденційності.

3.8 Приклади програмної взаємодії через API
Telegram надає відкритий Bot API, який дозволяє створювати ботів та автоматизовані сервіси.
Через API можна отримувати повідомлення користувачів, надсилати відповіді, працювати з файлами, кнопками, командами та каналами.
Це демонструє архітектурний принцип відокремлення клієнтського рівня від внутрішньої серверної реалізації через стандартизований інтерфейс.

3.9 Організація матеріалів у GitHub
Матеріали практичного заняття організовані у GitHub-репозиторії відповідно до вимог дисципліни.
3.10 Підготовка презентації та відеозапису
За результатами виконання практичного заняття було підготовлено презентацію та відеодоповідь, присвячені архітектурі Telegram.
У презентації висвітлено загальну характеристику системи, архітектурний стиль, основні компоненти, масштабування, безпеку та приклади роботи через API.
Відеодоповідь узагальнює ці положення у логічно послідовному вигляді. 

4 ВИСНОВКИ
У результаті виконання практичного заняття було досліджено архітектуру програмної системи Telegram як приклад сучасної платформи для обміну повідомленнями та передавання мультимедійного контенту великій кількості користувачів.
У ході роботи встановлено, що Telegram є багатокомпонентною розподіленою системою, архітектура якої поєднує клієнтський рівень, серверну частину, підсистеми доставки повідомлень, служби зберігання даних та механізми безпеки.
Для Telegram характерне використання клієнт-серверної архітектури з елементами мікросервісного підходу, що забезпечує гнучкість, високу доступність та можливість масштабування.
Окрему увагу приділено балансуванню навантаження, sharding, replication, CDN та використанню протоколу MTProto. Це дозволяє платформі стабільно працювати під великим навантаженням і забезпечувати швидкий доступ користувачів до сервісу.
Отже, виконання практичного заняття дало змогу сформувати практичні навички архітектурного аналізу складних програмних систем та поглибити розуміння сучасних підходів до побудови масштабованих цифрових платформ.

Приклад програмного коду
using Telegram.Bot;
using Telegram.Bot.Types;

var botClient = new TelegramBotClient("YOUR_BOT_TOKEN");

// Отримання оновлень
var updates = await botClient.GetUpdatesAsync();

foreach (var update in updates)
{
    if (update.Message?.Text != null)
    {
        // Відправка повідомлення
        await botClient.SendTextMessageAsync(
            chatId: update.Message.Chat.Id,
            text: $"Ви написали: {update.Message.Text}"
        );
    }
}
