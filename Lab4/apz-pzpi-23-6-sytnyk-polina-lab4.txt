4.2 Опис прийнятих інженерних рішень
У межах лабораторної роботи для програмної системи Click&Rent було реалізовано підхід до горизонтального масштабування серверної частини. Основною метою є забезпечення можливості обробки більшої кількості запитів користувачів без зміни бізнес-логіки програмної системи.
Система Click&Rent являє собою web-застосунок для оренди одягу та аксесуарів. Серверна частина реалізована на платформі ASP.NET Core MVC та використовує Microsoft SQL Server для зберігання інформації про користувачів, речі, оренди та адміністративні дані.
Для демонстрації можливості масштабування було підготовлено Docker-конфігурацію, що дозволяє запускати декілька екземплярів серверної частини та розподіляти навантаження між ними за допомогою балансувальника навантаження Nginx.
Для навантажувального тестування використовується інструмент Locust, який дозволяє моделювати одночасну роботу великої кількості користувачів та оцінювати продуктивність системи.
Схема масштабованого розгортання має наступний вигляд:
Користувач → Nginx Load Balancer → ASP.NET Core Server → SQL Server Database
Такий підхід дозволяє збільшувати кількість серверних екземплярів без внесення змін до структури програмної системи та забезпечує більш стабільну роботу при збільшенні навантаження.

4.3 Опис технічних рішень, які роблять масштабування можливим
Для реалізації масштабування серверної частини використовується Docker. Серверна частина Click&Rent упаковується у Docker-контейнер за допомогою Dockerfile. Після створення образу можна запускати декілька незалежних екземплярів застосунку.
Усі екземпляри сервера працюють з однією спільною базою даних Microsoft SQL Server. Це дозволяє забезпечити актуальність даних незалежно від того, який серверний екземпляр обробляє запит користувача.
Для балансування навантаження використовується Nginx. Усі зовнішні HTTP-запити надходять до Nginx, після чого перенаправляються до одного з доступних екземплярів серверної частини.
Docker Compose використовується для автоматичного запуску всіх компонентів системи. Конфігурація містить опис серверної частини, балансувальника навантаження та допоміжних сервісів.

За результатами навантажувального тестування було встановлено, що web-застосунок Click&Rent коректно обробляє запити до основних сторінок системи. Під час тестування виконувались переходи на головну сторінку, сторінку каталогу речей, а також сторінки жіночого та чоловічого одягу.
Середній час відповіді сервера залишався невеликим і складав близько 10 мс. Разом із тим під час тестування були зафіксовані помилки типу RemoteDisconnected, які виникали через передчасне закриття HTTP-з’єднання сервером. Отримані результати дозволили оцінити продуктивність системи та визначити напрями її подальшої оптимізації.

4.5 Аналіз вузьких місць під час навантаження
Для аналізу вузьких місць системи використовувалися інструменти моніторингу Docker та статистика Locust.
Основними ресурсами, що споживаються під час навантаження, є процесор, оперативна пам’ять та база даних Microsoft SQL Server.
У ході тестування було встановлено, що найбільше навантаження припадає на серверну частину застосунку та базу даних. При збільшенні кількості одночасних користувачів зростає кількість звернень до таблиць користувачів, речей та оренд.

Аналіз результатів показав, що основною проблемою під час тестування стали помилки типу RemoteDisconnected. Такі помилки виникають у випадках, коли сервер завершує з’єднання до надсилання повної відповіді клієнту. Причиною можуть бути особливості локальної конфігурації ASP.NET Core, налаштування HTTPS або параметри мережевої взаємодії.
Основним потенційним вузьким місцем у майбутньому може стати база даних Microsoft SQL Server, оскільки всі серверні екземпляри працюють зі спільним джерелом даних. Для підвищення продуктивності доцільно використовувати оптимізацію SQL-запитів, індексування таблиць та механізми кешування даних.
Отже, у ході лабораторної роботи було реалізовано підготовку системи Click&Rent до горизонтального масштабування, проведено навантажувальне тестування та проаналізовано поведінку системи під навантаженням.
