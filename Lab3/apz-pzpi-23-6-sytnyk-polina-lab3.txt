3.2 Опис прийнятих інженерних рішень
У межах лабораторної роботи було розроблено front-end частину програмної системи Click_Rent. Серверна частина системи була реалізована раніше, тому web-застосунок створено як клієнтську частину, що взаємодіє з уже наявною базою даних та backend-логікою. Основним завданням web-частини є забезпечення роботи користувачів у браузері, а також надання окремого адміністративного інтерфейсу для керування системою.
Web-застосунок реалізовано з використанням ASP.NET Core MVC та Entity Framework Core. Такий підхід було обрано через простоту реалізації архітектури Model-View-Controller, зручну інтеграцію з SQL Server та можливість швидко створювати веб-інтерфейси для роботи з даними.
У web-застосунку реалізовано два логічно відокремлені інтерфейси: користувацький та адміністративний. Звичайний користувач має доступ до перегляду каталогу речей, перегляду жіночих і чоловічих категорій, реєстрації та входу в систему. Адміністратор має доступ до окремої адміністративної панелі для керування клієнтами, речами, орендами, імпортом та експортом даних, а також створення резервних копій.
Для автентифікації реалізовано систему входу та реєстрації користувачів. Після входу користувач отримує доступ до відповідного інтерфейсу залежно від ролі. Адміністратор має розширені права доступу до керування даними системи.
Окремо реалізовано підтримку інтернаціоналізації та локалізації. Web-застосунок підтримує українську та англійську мови інтерфейсу. Також реалізовано відображення дати та часу відповідно до обраної локалі та підтримку сортування даних у каталозі.
3.3 Опис взаємодії компонентів web-застосунку одне з одним та з іншими елементами системи
Web-застосунок Click_Rent побудований як клієнтська частина системи оренди речей. Користувач працює з інтерфейсом через браузер, а всі операції виконуються через контролери ASP.NET Core MVC та взаємодію з базою даних SQL Server через Entity Framework Core.
Основна взаємодія починається з авторизації або реєстрації користувача. Після успішного входу користувач отримує доступ до функцій системи відповідно до своєї ролі. Звичайний користувач може переглядати каталог речей, виконувати пошук та перегляд категорій. Адміністратор отримує доступ до адміністративної панелі.
Користувацька частина web-застосунку взаємодіє з модулями каталогу, категорій речей та системою оренди. Коли користувач відкриває каталог, контролер отримує інформацію з бази даних через RentDbContext та передає її до представлень для відображення.
Адміністративна частина взаємодіє з таблицями користувачів, клієнтів, речей та оренд. Через web-інтерфейс адміністратор може переглядати інформацію про клієнтів, керувати речами, переглядати оренди, експортувати дані, імпортувати дані та створювати резервні копії.
Таким чином, web-застосунок не містить складної бізнес-логіки, а виконує роль зручного інтерфейсу для роботи із системою оренди речей. Контролери відповідають за обробку запитів, Entity Framework Core забезпечує взаємодію з базою даних, а представлення відповідають за відображення інформації користувачам.
3.4 UML діаграми

UML діаграма прецедентів
UML діаграма прецедентів web-застосунку відображає основні сценарії взаємодії користувача та адміністратора із системою Click_Rent. На діаграмі показано два основні типи акторів: клієнт і адміністратор. Клієнт може виконувати реєстрацію, вхід до системи, переглядати головну сторінку, каталог речей, жіночі та чоловічі категорії, змінювати мову інтерфейсу та сортувати товари. Адміністратор має додаткові можливості: перегляд адміністративної панелі, керування клієнтами, речами та орендами, а також виконання експорту, імпорту й резервного копіювання даних.

Діаграма дозволяє показати функціональний поділ web-застосунку на користувацьку та адміністративну частини. Такий поділ відповідає рольовій моделі системи, де клієнт працює з каталогом і категоріями речей, а адміністратор керує даними системи.
 

Рисунок 3.1 – UML діаграма прецедентів web-застосунку Click_Rent

UML діаграма компонентів
UML діаграма компонентів показує будову web-застосунку Click_Rent та його взаємодію з іншими частинами системи. На діаграмі відображено основні компоненти: користувацький web-інтерфейс, адміністративний інтерфейс, контролери, модуль локалізації, модуль експорту та імпорту даних, Entity Framework Core, RentDbContext, SQL Server та мобільний API.

Ця діаграма демонструє, що web-застосунок взаємодіє з базою даних через серверну частину та Entity Framework Core. Також показано, що мобільний застосунок може отримувати дані через окремий API-контролер. Такий підхід дозволяє розділити відповідальність між web-інтерфейсом, backend-логікою, базою даних та мобільним клієнтом.
 

Рисунок 3.2 – UML діаграма компонентів web-застосунку Click_Rent

UML діаграма пакетів
UML діаграма пакетів відображає логічну структуру web-застосунку Click_Rent. На діаграмі показано основні пакети проєкту: Controllers, Views, Models, Data, Services та wwwroot. Пакет Controllers містить контролери, які обробляють HTTP-запити. Пакет Views містить сторінки інтерфейсу користувача. Пакет Models описує сутності системи, такі як клієнти, речі, типи речей, оренди та користувачі. Пакет Data містить RentDbContext для взаємодії з базою даних. Пакет Services містить допоміжну бізнес-логіку.

Такий поділ дозволяє розділити відповідальність між частинами застосунку та спрощує подальший супровід і розвиток системи.
 

Рисунок 3.3 – UML діаграма пакетів web-застосунку Click_Rent

UML діаграма діяльності
UML діаграма діяльності відображає основний процес роботи користувача із web-застосунком Click_Rent. На діаграмі показано запуск сайту, виконання входу або реєстрації, визначення ролі користувача та перехід до відповідного інтерфейсу. Якщо користувач є клієнтом, він переходить до користувацького інтерфейсу, переглядає каталог, обирає категорії, сортує товари та змінює мову інтерфейсу. Якщо користувач є адміністратором, він переходить до адміністративної панелі та може керувати клієнтами, речами, орендами, експортувати й імпортувати дані, а також створювати резервну копію.

Діаграма дозволяє наочно показати різницю між сценаріями роботи звичайного користувача та адміністратора.
